Added Auto Entities List module — A new data module that dynamically discovers and lists Home Assistant entities by domain, device_class, state, and keyword filters, with four row styles (Compact, Detailed, Slim, Card), pinned and hidden entities, sort by name / last-changed / state / domain, configurable columns and row spacing, optional entity-color highlighting (e.g. RGB lights tint their own row), per-row label/icon/color overrides, and full action support (tap/hold/double-tap). Localized across all 13 supported languages.
Added dynamic icon and icon_color support to Text-module templates — Unified templates on Text modules can now return `icon` and `icon_color` in addition to `content`, `color`, and `container_background_color`, letting a single template drive both the displayed value and the icon glyph and color from one expression (e.g. swap between mdi:fire / mdi:thermometer / mdi:snowflake based on a temperature sensor). The flash-of-default-content cache from beta4 was extended to cover icon and icon_color too, so re-mounts paint the last-good icon instantly. New cheatsheet example, supported-keys table, and module docs updated.
Improved module picker layout with four themed groups — The "Add Module" picker now organizes modules into Data & Lists, Content & Media, Controls, and Input Helpers (each with its own header icon and description) instead of one long flat "Content Modules" list, making the growing module catalog much faster to scan. Group rendering is now shared via a single helper for consistent spacing and styling. Dynamic List was also recategorized from layout into the new Data & Lists group, where it fits the auto-list, alert-center, activity-feed, battery-monitor, and area-summary family.
